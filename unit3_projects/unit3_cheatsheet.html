SQL Cheat Sheet

Note: this cheat sheet lists common SQL commands (MySQL syntax) and concepts — this will be updated with time to become more complete. Thanks to Tali Herzka and Michael Yeung for collaboration!

SELECT Statements

To select ALL of the fields for entries in a table, use: 

                        SELECT * FROM table_name

To select a given field for entries in a table, use: 

                        SELECT field_name FROM table_name

To select multiple fields for entries in a table, use: 

                        SELECT f_name1, f_name2, f_name3 FROM table_name

To select data for entries in a table based on specific conditions, use: 

                        SELECT f_name FROM t_name
                        WHERE conditions

To order the result of a query based on ascending order, use: 

                        SELECT f_name FROM t_name
                        WHERE conditions
                        ORDER BY f_name ASC

To order the result of a query based on descending order, use: 

                        SELECT f_name FROM t_name
                        WHERE conditions
                        ORDER BY f_name DESC

To order the result of a query based on multiple fields, use: 

                        SELECT f_name FROM t_name
                        WHERE conditions
                        ORDER BY f_name1 DESC, f_name2 ASC

To select only unique values from a given column, use: 

                        SELECT DISTINCT f_name FROM t_name

To select only values from a given column that match a certain pattern (with a wildcard character '%'), use: 

                        SELECT f_name FROM t_name
                        WHERE f_name LIKE pattern

To select entries from a given column only if the entries are present in a list that you provide, use: 

                        SELECT f_name FROM t_name
                        WHERE f_name IN ('listmember1', 'listmember2', etc.)

To select only the values from a given column that fall within a provided range, use: 

                        SELECT f_name FROM t_name
                        WHERE f_name BETWEEN value_1 AND value2

JOINs

To 'inner join' tables based on JOIN conditions, use: 

                        SELECT * FROM t_name1
                        INNER JOIN t_name2 ON conditions

To join the right table to the left table, returning null for non-matching values, use: 

                        SELECT * FROM t_name1
                        LEFT JOIN t_name2 ON conditions

To join the left table to the right table, use: 

                        SELECT * FROM t_name1
                        RIGHT JOIN t_name2 ON conditions

To return all the entries from both tables, use: 

                        SELECT * FROM t_name1
                        FULL OUTER JOIN t_name2 ON conditions

Relationships

One to One Relationships

One record can belong to only one other record — vice versa. For example, imagine that my webapp had a users table. Also, each user could choose to connect his or her Facebook account to their profile on my webapp for extra perks. I might choose to have a facebook_accounts table — this would allow me to create a one to one relationship between users and facebook_accounts because each user can only have one Facebook account and each Facebook account can only belong to one user.

This relationship is created by adding a foreign key that maps the facebook_accounts table to the users table, usually through a facebook_account_id field in the users table that maps to the id field of the facebook_accounts table.

One to Many Relationships / Many to One Relationships

One record can have many other records associated with it, but those records can only belong to the one associated record. Imagine that your webapp has a users table. Each user can create lots of jokes to share on your webapp. So, you have a jokes table. Each user can have as many jokes as he or she wants. But, each joke belongs to a single user. This is a one to many relationship.

This relationships is created by adding a foreign key as a field on the 'many' table (in our example, the jokes table). The foreign key maps onto the primary key of the 'one' table (in our example, the users table). Usually, this is achieved by adding a user_id field to the jokes table that maps to the id field in the users table.

Many to Many Relationships

One record can have many other records associated with it, and those records can also belong to many other records. Imagine that you have an ecommerce webapp. You have a products table as well as an orders table. Each order can have many different products. Also, each product can be in many different orders. This is a many to many relationship.

This relationship is established by creating a hybrid table that can be used as a 'lookup' table. This table will contain two foreign keys — the id for the item and the id for the order — so that the hybrid table can join the items with the orders.

For example, you could create an items_orders table that has item_id and order_id columns. Then, anytime you needed to find the items in an order (or the orders associated with an item), your queries can access the 'lookup' table to join all the entries needed.
