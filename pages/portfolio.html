iswear

NPM Package | Source

iswear is an open-source utility library for working with Promises (the Promises/A+ specification). I built iswear in my fourth week studying at Hack Reactor. This library allows a developer to write asynchronous code as though it was synchronous (in appearance). For example, when working with Node, instead of having to pass around callbacks in an unorganized fashion, a developer could use iswear's .then and .catch to handle the result of invoking the asynchronous methods of Node.

Learning Objectives

  • Promises specification
  • General open-source library creation
  • Proxies
  • User-intent abstraction
  • NPM package publication

Lernhow

Demo | Source

Lernhow provides 5-step guides to knowledge. Don't search sprawling wikis for information on how to do something — come Lernhow.

Lernhow is a project I completed in a week sprint while a student at Hack Reactor. From ideation to launch, Lernhow was created by myself, Raghuvir Kasturi, and Eric Kennedy.   • Rigorous workflow standards
  • Clear documentation and turn-key setup for contributors

Linguist

Demo | Source

Linguist allows people all around the world to talk to each other in different languages by automatically translating messages for them. Our app speaks for them, in a language they don't.

Linguist was completed in a four-day sprint while I was a student at Hack Reactor. This project was given to myself, Omar Alvarez, and Eric Kennedy, as a legacy project — we extended our fellow student's codebase into a whole new product. Linguist was built with the follow technologies: Microsoft Translator API, GitHub API, MongoDB, Mongoose ORM, Node.js, Express, Socket.io, Backbone, Underscore, jQuery, Karma, Gulp, Bash, Windows Azure, and more.

Learning Objectives

  • Background workers, task runners, and shell scripting
  • Web-socket communication between server and clients
  • Consumption and integration of multiple external APIs
  • Construction of public emoji-fetcher API
  • Translation, internationalization, and multi-lingual compatibility

Shortenz

Source
